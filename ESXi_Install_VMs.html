Overview

In this section, you will install the Virtual Machines for ISEAGE and configure them.

Install Virtual Machines

In this section we will download and install the Virtual Machines images to build ISEAGE. All network connections should automatically be made if the network is configured correctly.

Note: It is highly recommended that you connection to your server be through a wired connection with a gigabit switch since the files to be transfered to the server are quite large.

Install the Virtual Machines

  1. First, unzip all of the virtual machine images. ESXi needs to images to be uncompressed.

  2. After unzipping the Virtual Machine images, we need to open up the datastore browser on ESXi. First, log into ESXi if not already, then go to Configuration then Storage and the right click on your datastore and then go to Browse Datastore...




    3. Figure 1: Browse Datastore


  3. The datastore browser will look similar to Figure 2. Next, make sure the root folder (/) of the datastore is highlighted and then click the upload button, and choose upload folder. Choose the folder that contains one of the Virtual Machines. Repeat this process for all of the Virtual Machines.




    4. Figure 2: Datastore Browser


  4. Note: It is possible to have multiple uploads running in parallel. This can substantially reduce the amount of time needed to upload the images.

  5. After uploading all of the VMs to the server, we need to add all of the VMs to ESXi. To do this, click on one of the VM folders. Then find the Virtual Machine file (ending in .vmx) and click it to highlight the file. After highlighting the Virtual Machine file, click on the button in the upper left corner of the Database Browser window to add the virtual machine to the inventory. A window will open. Leave the Virtual Machine name alone and click next. On the following window also click next. On the final window click finish. Repeat these steps for all of the Virtual Machines.




    6. Figure 3: Add VM to Inventory


  6. After adding all of the Virtual Machines to the inventory, we need to manually start up all of the Virtual Machines to allow ESXi to update there configuration information. Go to the Virtual Machines tab on the main ESXi window. Right-click on the first Virtual Machine and go to Open Console.




    7. Figure 4: Open VM Console


  7. In the VM console window, click the Green Play Button to start the Virtual Machine.




    8. Figure 5: Start the VM


  8. A window will pop up asking if you moved or copied the Virtual Machine. Check the Copied it option and then click OK. Repeat these last two steps for the rest of the Virtual Machines.




    9. Figure 6: VM First Boot Message


  9. At this point all of the Virtual Machines should be installed and configured correctly. Next, we are going to configure the auto-start feature of ESXi to start up the Virtual Machines in the correct order whenever the physical server is powered up.

    10. Configure Auto-startup

  10. In this section we will configure the Virtual Machines to automatically start up in the correct order when the physical server is started. Go to the Configuration tab, click on Virtual Machine Startup/Shutdown and then click Properties.




    11. Figure 7: VM Startup/Shutdown Configuration


  11. In the configuration window which opens up, check the two options in the upper left side of the window.




    12. Figure 8: VM Startup/Shutdown Options


  12. Now we will configure the order in which the Virtual Machines will start up. First find the Snowbank Virtual Machine and click it to highlight it. Now, repeatedly click on the Move Up button to move the Snowbank VM into the Automatic Startup section.




    13. Figure 9: Snowbank Autostart


    Continue this process for the rest of the VMs until the startup order is the same as in Figure 10.




    Figure 10: Startup Order
