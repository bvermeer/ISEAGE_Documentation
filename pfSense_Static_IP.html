Overview

This section is optional and is only required if you would like your instance of ISEAGE to have a static IP address. This can be handy when accessing the ISEAGE control webpage from and external network.

Configure Static IP on Snowbank

This section is optional and only required if a static IP address if required for ISEAGE. By default, ISEAGE is configured to automatically obtain an IP address on the External Network using DHCP.

Getting to Snowbank's Configuration Page

  1. The Snowbank virtual machine is running the pfSense firewall software which is configure through a locally hosted webpage. To access Snowbank's configuration webpage we will first start of the Gatekey Virtual Machine. First, go to the Virtual Machine tab in vSphere Client and right-click on Gatekey and go to Open Console.




    2. Figure 1: Open Gatekey Console


  2. A new window should pull up that allows you to interact with the Gatekey Virtual Machine. Most likely the screensaver will be up. Click anywhere on the screen and the desktop should appear momentarily. Click on the Firefox logo in the lower left-hand side of the screen to start up Firefox.




    3. Figure 2: Open Firefox


  3. The homepage on Gatekey is set to the ISEAGE control webpage. Go to the IP address of Snowbank (192.168.1.1) to access Snowbank's configuration page. The username and password should be automatically filled in. If not the username is admin and the password is iseage.




    4. Figure 3: Log into Snowbank


  4. After logging into Snowbank you will be brought to Snowbanks main configuration screen which will look similar to Figure 4.




    5. Figure 4: Snowbank Main Configuration Screen


  5. Now go to the Interfaces -> WAN menu.




    6. Figure 5: Snowbank navigate to WAN Configuration


  6. The WAN configuration page will look similar to Figure 6.




    7. Figure 6: Snowbank WAN Configuration


  7. Now change the IPv4 Configuration Type to Static.




    8. Figure 7: Static IPv4


  8. Scroll down to the Static IPv4 configuration section and enter the IP address and netmask (the dropdown box) you desire to Snowbank.




    9. Figure 8: Static IPv4 IP


  9. Click on the add a new one link under the IPv4 Upstream Gateway section to add the information for the upstream gateway. After entering the information, click on the Save Gateway button.




    10. Figure 9: Static Gateway


  10. Scroll down to the bottom of the screen and click the Save button.




    11. Figure 10: Save WAN Settings


  11. Now go to the System -> Routing menu.




    12. Figure 11: Snowbank Routing


  12. The pfSense system does not delete old gateways when adding a new one which can sometimes lead to routing issues. To fix this, we will manually delete the old gateway. Click the X to the right of the gateway you are not using. In my case, I wasn't using the WANGW gateway. A message will pop up asking if you are sure you want to delete the gateway. Click OK.




    13. Figure 12: Snowbank Gateways


  13. After delete the extra gateway, click the Apply changes button at the top of the screen.




    14. Figure 13: Apply Changes


  14. Congratulations! You have successfully configured Snowbank with a static IP address.
